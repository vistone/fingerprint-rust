# 配置文件目录

此目录包含项目的所有配置文件，按功能类别组织。

## 📁 目录结构

```
config/
├── build/          # 构建相关配置
├── deployment/     # 部署相关配置
├── monitoring/     # 监控相关配置
└── services/       # 服务相关配置
```

## 🎯 各子目录说明

### build/ - 构建配置
包含项目构建所需的基础配置文件：
- `Cargo.toml` - Rust工作区配置
- `rust-toolchain.toml` - Rust工具链版本指定

### deployment/ - 部署配置
包含各种部署环境的配置：
- `k8s/` - Kubernetes部署配置
- `docker/` - Docker容器配置
- `scripts/` - 部署脚本

### monitoring/ - 监控配置
包含系统监控和告警配置：
- `prometheus/` - Prometheus监控配置
- `grafana/` - Grafana仪表板配置
- `alertmanager/` - 告警管理配置

### services/ - 服务配置
包含各服务的独立配置：
- `ml-api/` - 机器学习API服务配置
- `gateway/` - API网关服务配置

## 📝 配置文件规范

### 命名约定
- 配置文件使用清晰的功能描述命名
- 环境特定配置添加环境后缀（如 `-prod`, `-dev`）

### 版本控制
- 所有配置文件都应纳入版本控制
- 敏感信息使用环境变量或secret管理
- 定期审查和更新配置文件

## 🔧 使用指南

### 添加新配置
1. 确定配置所属的功能类别
2. 在相应子目录中创建配置文件
3. 更新此README文档
4. 在项目文档中添加相关说明

### 修改现有配置
1. 创建配置变更提案
2. 经过团队评审后实施
3. 更新相关文档
4. 测试配置变更的影响

## ⚠️ 注意事项

- 不要在配置文件中硬编码敏感信息
- 保持配置文件的环境无关性
- 定期清理过时的配置文件
- 确保配置文件的向后兼容性
