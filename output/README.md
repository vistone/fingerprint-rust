# 输出文件目录

此目录用于集中管理项目运行时产生的所有输出文件。

## 📁 目录结构

```
output/
├── data/      # 数据输出文件
├── logs/      # 系统和应用日志
├── reports/   # 生成的报告文件
└── temp/      # 临时文件
```

## 🎯 各子目录说明

### data/ - 数据输出
存储程序运行过程中产生的数据文件：
- `dns/` - DNS相关数据和缓存
- `pcap/` - 网络数据包捕获文件
- `analysis/` - 分析结果和统计数据

### logs/ - 日志文件
包含系统和服务的日志信息：
- 应用程序运行日志
- 错误和警告信息
- 性能监控日志
- 访问日志

### reports/ - 报告文件
存储自动生成的各种报告：
- 性能测试报告
- 分析结果报告
- 系统状态报告
- 安全审计报告

### temp/ - 临时文件
用于存放临时性的中间文件：
- 缓存文件
- 临时计算结果
- 过渡数据文件

## 📝 文件管理规范

### 命名约定
- 使用有意义的文件名前缀
- 添加时间戳以便追溯（格式：YYYYMMDD_HHMMSS）
- 按功能或类型使用子目录组织

### 生命周期管理
- **临时文件**: 定期清理（建议每日）
- **日志文件**: 按大小或时间轮转
- **数据文件**: 根据业务需求保留适当期限
- **报告文件**: 永久保存重要报告

### 存储策略
```
日志文件: 保留最近30天
临时文件: 保留最近7天  
数据文件: 根据业务需求确定
报告文件: 重要报告永久保存
```

## 🛠️ 管理工具

### 查看当前状态
```bash
# 查看目录使用情况
du -sh output/*

# 查看最新生成的文件
find output/ -type f -mtime -1

# 统计各类文件数量
find output/ -name "*.log" | wc -l
```

### 清理操作
```bash
# 清理旧的临时文件
find output/temp/ -mtime +7 -delete

# 压缩旧日志文件
gzip output/logs/*.log.1
```
